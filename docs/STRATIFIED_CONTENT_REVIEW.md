# 分层内容审查协议

这套协议把内容优化拆成可复现的闭环：固定输入矩阵生成完整人生，自动审查找高证据问题，人工通读判断文学质量，修订后用相同 seed 回归，再换 seed 防止过拟合。

## 为什么不能只随机跑

均匀随机开局并不等于内容覆盖。家庭阶层选项随年代和城乡变化，人口较少的地区、女性乡村高资源家庭、近代城市低资源家庭等组合很容易在随机样本中缺席。分层矩阵保证每次回归都能看见同一组人物横截面。

`balanced` 是日常基线，`full` 用于大版本发布前的深测。大量新增事件后，先用旧 seed 比较回归，再换一个 seed 复验；不要根据某一个随机批次反复调权重。

## 自动证据的置信层级

### 阻断项

- CSV、manifest 或关联键损坏。
- 实际触发违反事件自身声明的年龄、年份、出生年份、性别、阶层或地区条件。
- 男性角色被叙述为怀孕、分娩等明确身体冲突。
- 玩家当年看到尚未出现的产品或社会词汇，例如 1996 年出现“家长群”。
- 全日制在读与在职身份无声明地重叠，教育层级倒退，或主要身份与教育/职业状态不一致。
- 已经有职业履历的人再次触发“第一份工作”，或就业后无离职、停薪/非全日制等桥接事件便重新成为全日制新生。

### 人工审查队列

- 文案词义提示年龄不合常理。
- 0—3 岁正文出现成年人的判断、盘算或稳定自传式记忆；幼年应优先采用身体感觉、照料者动作和家庭转述的外部镜头。
- 结婚、生育、退休叙述没有进入相应状态。
- 强地点纹理与所在地疑似不符。
- 同一事件覆盖过多人，或高频事件只有一段固定文案。
- 某个分群主要依赖“平常一年”。
- 近代出生群体几乎没有早亡，或大量人物在同一年龄死亡。

人工队列只说明“值得读原文”，不等于程序已经证明内容错误。例如成年人提到孩子的襁褓，不应被误判为成年人仍在襁褓；因此年龄规则尽量匹配“你”本人的动作，并保留样本文本供复核。

## 高频问题的三个不同口径

1. 事件占比：该事件在全部个人年中占多少。
2. 人生渗透率：至少出现一次该事件的人生占多少。
3. 单局复现率：同一人生里重复出现同一事件或同一文案多少次。

一个 `maxOccurrences: 1` 的事件可以没有单局重复，却仍以 90% 的人生渗透率制造“人人都经历同一件事”的模板感。反过来，感冒等事件多次发生并非不真实，但如果每次用同一句话，文学时间线仍会露出机械结构。三个口径必须同时看。

## 人工通读抽样

每轮至少从审查 JSON 的以下分群各抽一局完整通读：

- 八个出生世代。
- 男女、城乡、低中高家庭资源。
- 华北/东北/沿海/中部/西南/西北/边疆与特殊历史地区。
- 脆弱、均衡、韧性属性。
- 最早死亡、最长寿、平常年最多、头部事件最多的极端局。

逐年检查：事件是否可能在当时当地发生；年龄和既有状态是否承接；同一类命运是否因人物不同而换了观察角度；幽默是否来自生活本身而非段子口吻；删去时代名词后，句子是否仍有一个具体的人。

## 大规模扩库时的门禁顺序

1. 先过数据定义与固定年谱校验。
2. 跑 `smoke`，处理阻断项。
3. 跑固定 seed 的 `balanced`，比较频率、分群覆盖、人生状态转移和死亡分布。
4. 人工通读高风险样本并修订文案或条件。
5. 换 seed 再跑 `balanced`。
6. 发布前跑 `full`；规模增大后可提高 `replicates`，但不能用局数替代分层覆盖。

事件库迈向数万条时，事件定义覆盖率会自然下降，不能简单要求每批触发全部事件。更重要的是每个分群拥有足够多可竞争的具体事件、头部渗透率下降、固定文案支配率下降，并且新增内容没有制造时代与状态冲突。
