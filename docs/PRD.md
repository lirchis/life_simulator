# 人生模拟器 PRD

## 1. 项目概述

### 1.1 一句话定位

一款轻量、可反复游玩的文本人生模拟器。玩家在开局选择特质与属性，之后通过“下一年”按钮推进人生，每年触发随机事件、改变属性、资源、特质与标签，最终生成一份带有荒诞感与现实共鸣的人生报告。

### 1.2 产品目标

- 做出一个无需复杂教学、打开就能玩的一局制人生模拟器。
- 让玩家在第一局内理解：开局参数会影响事件，事件会影响人生，人生会导向不同结局。
- 建立可扩展的事件、开局特质、结局数据结构，后续主要通过补内容扩展玩法。
- 支持多周目重玩，通过结局、称号、事件图鉴提升留存。

### 1.2.1 体验原则

- 开始要快：从首页到第一年事件尽量少步骤，手选开局可用，但必须提供一键随机和默认推荐值。
- 展示要轻：不做复杂仪表盘、长说明和大段教学，事件文本和少量变化摘要是主角。
- 信息密度要高：同一屏内尽量展示年龄、年份、阶段、事件、关键数值变化和下一步操作，减少无意义留白。
- 操作要顺：完成开局后，核心路径只能是连续点击“下一年”，中途不得要求玩家做任何选择。
- 反馈要短：数值变化、特质变化、标签变化和自动结果都用紧凑摘要呈现。

### 1.3 核心乐趣

- 随机命运：每一年都可能出现意料之外的事件。
- 参数反馈：玩家能感受到开局特质和属性影响了人生走向。
- 文本反转：事件文本要有现实共鸣，也要允许荒诞幽默。
- 结局收集：死亡、成功、失败、奇遇都可以成为可收藏结局。
- 快速重开：一局 3 到 8 分钟，失败也能马上再来。

### 1.4 首版不做

- 不做复杂开放世界、地图移动、实时战斗或大型 RPG 系统。
- 不做重度经营，不要求玩家每年分配大量资源。
- 不做账号、付费、广告、联网排行榜。
- 不依赖 AI 生成主流程。AI 可以作为后续增强项，用于生成总结、传记或分享文案。
- 不做冗长新手教程、低密度欢迎页、装饰性大图和复杂动画。

## 2. 核心玩法

### 2.1 核心循环

```text
开局选择参数
  -> 点击下一年
  -> 触发年度事件
  -> 属性 / 资源 / 特质 / 标签变化
  -> 解锁阶段事件
  -> 死亡或特殊结局
  -> 生成人生报告
  -> 再来一局
```

每局人生需要生成并展示固定随机种子。玩家可以复制种子分享给别人，复用同一套初始状态和 seed 时应尽量复现同一条人生随机序列。

### 2.2 玩家流程

1. 进入游戏首页，点击“开始新人生”。
2. 系统随机出现一组开局特质，玩家选择 3 个。
3. 玩家分配初始属性点，或选择“一键随机”。
4. 系统根据属性、特质和出生年份生成出生背景。
5. 玩家进入人生时间线。
6. 每点击一次“下一年”，年龄增加 1 岁，并触发 1 到 3 条事件。
7. 关键事件可以包含多个自动结果，引擎根据人物状态、权重和 seed 决定走向，并直接展示结果。
8. 当生命结束或触发特殊结局时，展示人生报告与可分享摘要。

### 2.3 开局参数

| 类型 | 说明 | 首版建议 |
| --- | --- | --- |
| 基础属性 | 影响事件触发、事件结果和结局评分 | 体质、智力、魅力、家境、运气、心态 |
| 开局特质 | 玩家开局选择，UI 可继续称为“天赋”；进入状态后统一写入 `traits` | 随机 10 选 3 |
| 出生背景 | 由出生年份、性别、地区、家庭阶层、家境、运气和特质共同生成 | 性别、历史省份、城市层级、家庭阶层、家庭资源标签 |
| 特质 | 相对稳定的个人特征，可来自开局、事件或年度自然变化 | 应试天分、虚弱体质、数字原住民、市场嗅觉 |
| 标签 | 记录经历、身份和社会状态，用于后续事件条件判断 | 大学经历、已婚、创业路线、土改受益、迁移经历 |

首版允许玩家手选初始状态，包括出生年份、性别、出生省份、城市层级、户口、家庭阶层、基础属性和开局特质。为了保持快速开局体验，同时提供“一键随机”能力。

出生省份、城市层级、家庭阶层需要符合出生年份的时代背景。玩家看到当年语境下的名称，例如“西康”“广东”“佃户”“干部子弟”，系统内部再映射到稳定的计算 code，用于事件概率和地区聚合。若历史省份后来拆到多个当代省份，系统按历史权重随机归属，而不是平均随机。

省份第一版全量支持，省份 code 使用拼音。极端事件和早死事件不提供单独跳过开关。

### 2.4 年度事件

年度事件是游戏主体。每一年推进时，系统先执行年度自然变化，再根据当前年龄、属性、资源、特质、标签、历史事件，从事件池中筛选并抽取事件。

年度自然变化不占事件位，也不弹成单独事件卡。若自然变化导致数值或特质变化，例如健康自然恢复 `+1`、年龄增长导致健康 `-1`、长期低健康获得“久病体弱”，时间线应在该年份下显示一条轻量“自然流变”摘要，让玩家知道状态不是凭空变化。

事件类型：

- 普通事件：每年出现，影响较小，负责塑造人生质感。
- 阶段事件：只在特定年龄段出现，如入学、高考、就业、退休。
- 关键事件：影响较大，可包含自动分叉结果，如职业去向、婚恋、创业、重大疾病。
- 连锁事件：某事件写入特质、标签或后续事件计划，后续几年可能触发后续事件。
- 死亡事件：健康、年龄、事故、极端选择等触发。
- 结局事件：满足特殊条件时触发，如“青年创业暴富”“终身隐居”“负债累累”。

同一个日常过渡事件可以拥有多种文本表达。若事件本质相同，只是因为时代、省份、阶层、城乡、性别或特质不同而需要不同叙述，优先作为同一个事件的多表达处理，例如“平常一年”“入学”“夜里发烧”“考得不错”。若它的触发概率、效果、后续依赖或历史意义已经明显不同，则拆成独立事件，例如“分到土地”“恢复高考”“被拉壮丁”。

### 2.5 无中途选择原则

开局完成后，玩家唯一的推进操作是“下一年”。升学、职业、婚恋、迁移、创业和危机处理等人生分叉均由引擎自动决定，不允许弹窗、选项按钮或任何阻断推进的二次确认。

自动结果规则：

- 一个关键事件可以配置多个 `outcomes`。
- 引擎先按人物状态过滤可用结果，再结合权重和 seed 抽取一个结果。
- 最终结果及影响直接写入当年事件卡，玩家不需要额外操作。
- 相同开局状态和 seed 应复现相同的自动人生走向。

## 3. 人生阶段

| 阶段 | 年龄 | 重点事件 |
| --- | --- | --- |
| 幼年 | 0-6 | 家庭、健康、启蒙、意外、性格底色 |
| 学生 | 7-18 | 学习、同伴、兴趣、升学、青春期 |
| 青年 | 19-35 | 大学、职业、恋爱、城市选择、收入起步 |
| 中年 | 36-59 | 事业、家庭、房产、压力、健康、父母养老 |
| 老年 | 60+ | 退休、疾病、回忆、家庭关系、最终结局 |

## 4. 属性与资源

### 4.1 基础属性

| 属性 | 含义 | 影响 |
| --- | --- | --- |
| 体质 | 身体基础与抗风险能力 | 疾病、运动、寿命、意外恢复 |
| 智力 | 学习、判断、技术能力 | 升学、职业、创业、关键节点结果 |
| 魅力 | 外貌、表达、人际吸引力 | 社交、恋爱、职场机会、公众事件 |
| 家境 | 家庭资源与托底能力 | 出生背景、教育机会、购房、风险缓冲 |
| 运气 | 随机收益与黑天鹅规避 | 稀有事件、事故概率、财富波动 |
| 心态 | 情绪稳定、抗压与幸福感 | 压力事件、抑郁风险、家庭关系、最终幸福评分 |

### 4.2 派生资源

| 资源 | 说明 |
| --- | --- |
| 财富 | 由职业、家境、投资、负债事件影响 |
| 健康 | 由体质、年龄、医疗资源、生活事件和年度自然变化影响，过低会增加死亡概率 |
| 幸福 | 由心态、关系、成就、财富和重大遗憾共同影响 |
| 成就 | 由教育、职业、作品、创业、社会影响等事件累积 |
| 关系 | 包括亲情、友情、伴侣、子女等，首版可以先抽象为数值 |

### 4.3 数值范围建议

- 基础属性：0 到 10。
- 派生资源：0 到 100。
- 初始可分配属性点：20 点。
- 单项属性初始上限：10。
- 属性变化应尽量小，普通事件通常 `-1`、`+1`，关键事件可以 `-3` 到 `+3`。

## 5. 开局特质系统

### 5.1 特质规则

- 开局随机展示 10 个特质卡。为了玩家理解，UI 可继续使用“天赋”作为入口名称，但数据和引擎统一视为 `traits`。
- 玩家选择 3 个特质。
- 玩家拥有固定特质预算。强正面特质消耗更多点数，负面特质返还点数，双刃剑特质成本较低。
- 特质预算和特质池需要随出生年代变化。当前实现为：1900-1948 预算 2、1949-1977 预算 2、1978-1999 预算 3、2000-2020 预算 4。
- 只有“选满 3 个且总成本不超过预算”时才能开始，避免最优策略变成无脑选择全部正面特质。
- 特质分为普通、稀有、史诗。
- 特质可以直接改变初始属性，也可以通过底层特质影响事件权重。
- 特质允许有负面效果，最好存在“看起来很强但有代价”的双刃剑。
- 特质可以存在时代外显名和底层特质。例如“科举苗子”“旧学根底”“考试机器”都会写入底层特质“应试天分”，事件优先读取底层特质，而不是绑定某个时代词。
- 事件和年度自然变化也可以让主角获得新特质，例如长期低健康获得“久病体弱”。

### 5.2 特质类型

| 类型 | 示例 | 设计作用 |
| --- | --- | --- |
| 正面特质 | 早慧、强运、社交达人、互联网原住民 | 提供爽感，让玩家看到明显差异 |
| 负面特质 | 玻璃心、体弱、成分阴影、房价压顶 | 制造挑战和戏剧性 |
| 双刃剑 | 卷王、敏感艺术家、敢下海、补习班老兵 | 提供策略感和不确定性 |
| 时代特质 | 宗族庇护、票证脑子、下岗阴影、小镇做题家 | 强化历史代入和时代辨识度 |
| 性别与时代特质 | 女校机会、长女当家、家中长子、铁姑娘、兵役体格、理工少女 | 表达性别在不同时代中的制度机会、家庭期待和劳动分工 |
| 底层特质 | 应试天分、数字原住民、市场嗅觉、求生本能 | 供事件系统稳定引用，避免绑定时代外显名 |

## 6. 事件系统

### 6.1 事件筛选逻辑

事件不是完全随机，而是在年龄、年份、地区、属性、资源、特质、标签和历史经历的共同约束下随机。

推荐流程：

1. 根据年龄筛选可触发事件池。
2. 根据硬条件过滤，例如智力大于等于 7、拥有“大学经历”标签、拥有“应试天分”特质。
3. 根据权重抽取事件，权重受属性、资源、特质、标签、历史事件、地区和跨时间 modifier 影响。
4. 执行事件效果，更新属性、资源、特质、标签、关系和事件日志。
5. 判断是否触发死亡、阶段转折、成就或结局。

### 6.2 事件数据结构草案

```json
{
  "id": "gaokao_breakthrough",
  "title": "高考超常发挥",
  "category": "school",
  "ageRange": [17, 19],
  "yearRange": [1977, 2020],
  "baseWeight": 30,
  "conditions": {
    "all": [
      { "path": "attrs.intelligence", "gte": 6 },
      { "path": "attrs.mental", "gte": 4 }
    ]
  },
  "weightModifiers": [
    { "hasTrait": "exam_aptitude", "multiply": 1.8 },
    { "path": "education.score", "gte": 60, "multiply": 1.4 }
  ],
  "text": "你在高考中超常发挥，考上了一所不错的大学。",
  "effects": [
    { "path": "resources.happiness", "add": 5 },
    { "path": "resources.achievement", "add": 10 },
    { "addTag": "college" }
  ],
  "outcomes": []
}
```

### 6.4 条件操作符

首版支持这些条件操作符即可：

| 操作符 | 含义 |
| --- | --- |
| `eq` | 等于 |
| `neq` | 不等于 |
| `gt` | 大于 |
| `gte` | 大于等于 |
| `lt` | 小于 |
| `lte` | 小于等于 |
| `hasTag` / `missingTag` | 拥有 / 不拥有某经历标签 |
| `tagIn` / `tagNotIn` | 标签命中 / 不命中任一项 |
| `hasTrait` / `missingTrait` | 拥有 / 不拥有某特质 |
| `traitIn` / `traitNotIn` | 特质命中 / 不命中任一项 |
| `past` | 判断过去某年龄段或年份段是否满足条件 |

## 7. 结局与报告

### 7.1 结局类型

- 普通死亡：自然死亡、疾病、意外。
- 职业结局：普通职员、行业专家、创业者、艺术家、自由职业者。
- 财富结局：负债累累、财务自由、阶层跃迁、一夜返贫。
- 关系结局：家庭美满、孤独终老、复杂亲密关系。
- 荒诞结局：玄学暴富、离奇隐居、连续踩中时代风口。
- 特殊成就结局：科学家、作家、企业家、慈善家、传奇人物。

### 7.2 人生报告内容

人生结束后展示：

- 最终年龄。
- 死亡原因或结局原因。
- 最高成就。
- 关键人生节点时间线。
- 最终属性和资源评分。
- 解锁称号。
- 一段可分享的人生摘要。

### 7.3 评分维度

| 维度 | 说明 |
| --- | --- |
| 财富 | 最终财富资源 |
| 幸福 | 心态、关系、遗憾综合 |
| 成就 | 职业、作品、社会影响 |
| 健康 | 寿命与晚年健康 |
| 传奇度 | 稀有事件、特殊标签、反常规经历 |

## 8. 前端界面

首版 Web 需要同时适配桌面和手机端。手机端不是桌面版缩小，而是独立优化的信息层级和操作节奏。

### 8.1 页面模块

| 页面 | 功能 |
| --- | --- |
| 首页 | 游戏标题、开始按钮、继续/图鉴入口 |
| 开局特质选择页 | 随机特质池、稀有度、选择限制、刷新机制 |
| 属性分配页 | 属性点分配、一键随机、确认出生 |
| 人生页 | 年龄、核心属性、年度事件卡片、下一年按钮 |
| 报告页 | 时间线、最终属性、称号、结局、分享文案 |
| 图鉴页 | 已解锁特质、事件、结局和称号 |

### 8.2 人生页信息层级

优先展示：

1. 当前年龄。
2. 今年发生了什么。
3. 属性或资源变化。
4. 自动结果及影响。
5. 下一年按钮。

不建议在主界面堆太多复杂面板，否则会削弱“点点点”的流畅感。

人生推进页采用同屏连续时间轴，而不是每年切换一个新页面。玩家点击“下一年”后，新年份事件追加到当前时间轴底部，页面自然向下滚动或轻微跟随最新事件。年与年之间保持轻量分隔，让玩家像阅读一份持续生长的人生日志。

### 8.3 手机端体验要求

手机端是首版核心场景之一，必须看起来舒服、点起来顺手。

原则：

- 单列布局，不横向滚动。
- 主要操作放在底部安全区上方，例如“下一年”“开始这一生”“再来一局”。
- 人生推进页优先展示事件文本，状态面板默认折叠或放在事件之后。
- 年度事件在同一个时间轴内连续追加，不使用整页切换。
- 顶部只保留年龄、年份、阶段、当前地区和 seed 复制入口，不堆完整状态。
- 开局设置必须分步完成，每步只处理一组信息。
- 出生年份、省份、城市层级、户口、家庭阶层使用适合手机的控件。
- 属性分配使用大触控区域的 slider / stepper，避免小输入框。
- 特质卡在手机端单列展示，每张卡高度稳定，不因为文案长短跳动。
- 人生推进途中不出现选择弹层、确认弹窗或其他阻断操作。
- 报告页首屏必须看到结局标题、年龄/原因、摘要、再来一局和复制 seed。
- 极端死亡事件不额外遮挡，直接展示报告页。

移动端验收：

- 在 `390 x 844` 视口下无横向滚动。
- 主要按钮不被浏览器底部工具栏遮挡。
- 事件卡文字不溢出，不压住下一张卡。
- 年份之间的分隔轻量，不打断连续阅读。
- 顶部状态栏不超过两行。
- 单手连续点击“下一年”不需要反复移动手指到屏幕顶部。

## 9. MVP 范围

### 9.1 必须包含

- [x] 开局随机特质池，支持选择 3 个特质。

- [x] 年度推进按钮，每次推进年龄加 1。
- [x] 年度自然变化层，支持病后恢复、年龄健康衰退、低健康特质和轻量变化摘要。
- [x] 事件筛选、权重抽取、属性变化、特质写入、标签写入。

- [ ] 死亡判断与结局生成。
- [ ] 人生报告页，展示时间线、属性、称号与总结。

## 12. 待讨论问题

下一轮建议优先拍板：

1. 整体文案更偏现实扎心、荒诞搞笑，还是二者混合？
2. 是否明确做“中国式人生”语境，例如高考、户口、房价、职场、养老？
3. 自动结果是否需要更强的可解释性，例如在结果文案里暗示属性或特质的影响？
4. 死亡是否允许很早发生？如果允许，是否提供快速重开补偿？
5. AI 能力放在哪个版本？当前建议先放在 v0.3 之后，用于生成结局传记和分享文案。
