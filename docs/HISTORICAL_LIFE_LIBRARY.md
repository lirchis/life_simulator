# 隐藏人物年谱库

这套库不是“名人彩蛋事件”，而是一批可以逐年播放的历史文学作品。玩家满足一组出身条件后，只以一定概率进入某部作品；产品界面、人生文案和复制文本均不显示原型姓名。

## 不可破坏的规则

1. 每部作品必须从出生年写到死亡年，连续年份恰好一条，不能只存关键节点。
2. 命中后关闭普通事件、自动结果和自然流变；每次“下一年”只播放年谱中该年的固定条目。
3. 中途没有选择、职业确认或路线分叉。玩家开局后的唯一操作仍是“下一年”。
4. 触发条件至少包含精确出生年、性别、地域、城乡与家庭阶层，并可加属性门槛和命中概率。
5. 史料明确的年份写具体行动；史料稀少的年份写可证实的成长环境与时代背景，不凭空制造会面、对话或心理独白。
6. 写成一生，而不是写成颂词。功绩、信念、误判、权力及其社会后果都要进入同一条因果链。
7. 幽默只能来自生活的纹理、制度摩擦和人物处境；战争、饥荒、迫害与死亡处保持克制。
8. `sourceSubject`、研究来源和身份屏蔽词只属于资料层，不得写入玩家状态、标签、报告或剪贴板文案。

## 数据结构

每个人物一个文件，统一从 `src/data/historicalLives/index.js` 注册：

```js
{
  id: "chronicle_region_year_variant",
  sourceSubject: "仅供编辑部研究",
  matchChance: 0.25,
  selectionWeight: 1,
  trigger: {
    birthYearRange: [1893, 1893],
    genders: ["male"],
    provinces: ["hunan"],
    cityTiers: ["village"],
    hukou: ["rural"],
    familyClasses: ["rich_peasant"],
    attrs: { intelligence: { gte: 5 } },
  },
  hiddenTerms: ["真实姓名及常用称谓"],
  research: { note: "资料口径", sources: ["https://..."] },
  timeline: [
    { year: 1893, title: "……", category: "birth", text: "……", effects: [] },
    // 每年一条，直到含 die effect 的最后一年
  ],
}
```

`npm run check` 会逐部强制播放两遍，并验证：年份连续、每年唯一、无 choices/outcomes、无自然流变、不同随机种子产生完全相同的玩家文案、结尾明确死亡、隐藏身份词不泄露。

## 编辑流程

1. 先做资料卡：精确年份、地点、家庭、教育、职业、关系、疾病、争议与至少两类来源。
2. 再写逐年骨架，先保证出生至死亡没有断年。
3. 把关键事件写成因果节点，把空白年写成时间如何在身体、家庭和处境上经过。
4. 单独审查地名、机构名和政治词是否在当年已经存在。
5. 单独审查“上帝视角”：角色当年不能知道后来才发生的结论；可以由叙述者提示后果，但要清楚区分。
6. 用两枚不同 seed 强制播放并比较全部可见文本，确保作品没有随机分叉。
7. 通读复制出的完整一生，删掉连续使用同一种句式、空泛总结和硬塞的金句。
