# 人生模拟器 Web 首版 UI 设计方案

## 1. 设计目标

首版 Web UI 的核心是让玩家快速进入“开局设定 -> 一年一年推进 -> 看命运变化 -> 生成报告 -> 再来一局”的循环。

设计关键词：

- 快速：开局参数很多，但不能像填表一样累。
- 清晰：玩家随时知道自己几岁、发生了什么、状态怎么变了。
- 有戏：事件出现要有一点命运揭晓感。
- 可重开：死亡或结局后，报告页要推动玩家再来一局。
- 可分享：固定 seed 要有明显入口。
- 轻量：少装饰、少解释、少阻塞，事件文本和变化摘要优先。
- 高密度：同屏承载更多有效信息，避免大面积无意义留白。

首版 UI 优先级：

1. 快速开始。
2. 年度事件连续阅读。
3. 紧凑状态反馈。
4. 再来一局和 seed 分享。

后置内容：

- 大幅插画。
- 复杂动效。
- 大面积品牌页。
- 需要学习成本的高级统计面板。

## 2. 信息架构

```text
首页
  -> 新人生设置
    -> 出生设置
    -> 属性分配
    -> 天赋选择
    -> 确认开局
  -> 人生推进页
    -> 年龄 / 年份 / 阶段
    -> 状态摘要
    -> 事件卡片
    -> 下一年
  -> 人生报告页
    -> 结局
    -> 关键时间线
    -> 最终状态
    -> 称号
    -> seed 分享
    -> 再来一局
  -> 图鉴页
```

## 3. 视觉方向

### 3.1 整体风格

不做花哨游戏大大厅，做成“命运档案 + 年度日志”的感觉。界面应该像一个可以连续翻阅的人生记录器，有一点正式，又允许荒诞事件突然出现。

展示要轻量，信息密度要高。首页不做冗长 landing page；人生推进页不做大号装饰卡和复杂仪表盘；年度事件、年龄年份、变化摘要和下一年按钮必须始终是最显眼的内容。

推荐气质：

- 背景：浅灰白或低饱和纸张色。
- 主色：深墨绿或深蓝灰，用于按钮和关键状态。
- 辅色：琥珀色用于命运、稀有、警告；红色只用于死亡和重大损失。
- 字体：系统字体优先，中文阅读要舒服。
- 形状：卡片圆角 8px，按钮圆角 8px，不做过度圆润。

### 3.2 色彩 Token

```css
:root {
  --bg-page: #f6f3ed;
  --bg-panel: #fffdf8;
  --bg-muted: #ebe6dc;
  --text-main: #25231f;
  --text-muted: #756f65;
  --border: #d8d0c3;
  --primary: #26483f;
  --primary-hover: #1d3932;
  --accent: #b7791f;
  --danger: #a53a2a;
  --success: #3f7f58;
}
```

注意：这不是最终视觉稿，只是设计起点。首版避免整站单一深色或纯紫蓝渐变。

## 4. 页面设计

## 4.1 首页

目标：3 秒内让玩家知道这是什么，并开始。

首页必须轻量，不做长篇介绍。首屏要直接看到“开始新人生”和“一键随机开局”。如果有 seed 输入，也放在首屏，不进入二级页面。

布局：

- 顶部：游戏名、图鉴入口。
- 主区域：大标题“人生模拟器”，副标题展示一句随机命运文案。
- 主操作：开始新人生。
- 次操作：输入 seed / 继续上次 / 查看图鉴。

首屏文案示例：

- “选择起点，然后一年一年活下去。”
- “命运会记账，也会开玩笑。”
- “同一个 seed，可能是一条完全可复盘的人生。”

组件：

- `PrimaryButton`: 开始新人生
- `SeedInput`: 输入 / 粘贴 seed
- `IconButton`: 图鉴

## 4.2 新人生设置页

建议使用分步流程，不要一屏塞满。

默认状态要已经可开始。玩家可以精调参数，但不能被迫填完整表单。每一步都提供随机按钮，最终确认页可以一键开始。

步骤：

1. 出生设置
2. 属性分配
3. 天赋选择
4. 确认开局

页面结构：

```text
左侧：步骤导航 / 当前 seed
右侧：当前步骤表单
底部：上一步 / 随机一下 / 下一步
```

移动端：

- 步骤导航变成顶部进度条。
- 表单单列。
- 底部操作吸底。

### 4.2.1 出生设置

字段：

- 出生年份：`1900-2020`，使用数字输入 + 滑杆。
- 出生省份：下拉选择，code 用拼音，展示中文名。
- 城市层级：村庄 / 乡镇 / 县城 / 普通城市 / 二线 / 一线。
- 户口：农村 / 城镇，使用 segmented control。
- 家庭阶层：贫困 / 工薪 / 中产 / 富裕 / 精英。

体验规则：

- 每个字段都有随机按钮。
- 页面有“一键随机出生”。
- seed 固定时，随机结果可复现。

### 4.2.2 属性分配

属性：

- 体质
- 智力
- 魅力
- 家境
- 运气
- 心态

交互：

- 总点数：20。
- 单项范围：0-10。
- 使用 stepper 或 slider，不要只用输入框。
- 剩余点数固定显示。
- 点数不足时禁用增加按钮。

布局：

```text
属性名 | 当前数值 | - slider +
```

状态反馈：

- 0-3：偏弱
- 4-6：普通
- 7-8：优秀
- 9-10：极强

### 4.2.3 天赋选择

规则：

- 随机出现 10 个天赋。
- 选择 3 个。
- 显示天赋预算，例如“点数 2/3”。
- 天赋卡显示“消耗 2”“返还 1”或“免费”。
- 超出预算时禁用开始按钮，并给出轻量提示。
- 支持刷新天赋池。

卡片字段：

- 天赋名
- 稀有度
- 点数成本
- 简短描述
- 正负效果提示

稀有度视觉：

- 普通：灰色边框
- 稀有：绿色边框
- 史诗：琥珀色边框

不要把天赋卡做得太大。首版重点是快速扫读。

### 4.2.4 确认开局

展示：

- 出生年份、省份、城市层级、户口、家庭阶层
- 六项属性雷达或紧凑条形图
- 已选天赋
- seed

操作：

- 开始这一生
- 返回修改
- 复制 seed

## 4.3 人生推进页

这是最核心页面。

信息密度要求：

- 同一屏至少能看到当前年龄年份、1-2 条事件卡、核心状态摘要入口和下一年按钮。
- 事件卡高度克制，正文控制在 1-3 行优先。
- 数值变化用紧凑 pill 展示，不用大表格。
- 状态摘要只展示最关键 4-6 个值，完整属性折叠。
- 年份分隔符不超过一行。

桌面布局：

```text
顶部状态栏：
年龄 / 当前年份 / 人生阶段 / seed

主区域：
年度事件卡片列表

右侧栏：
核心状态摘要
标签 / 关系 / 教育 / 职业

底部：
下一年按钮
```

移动端布局：

```text
顶部状态栏
状态摘要折叠面板
事件卡片
下一年吸底按钮
```

### 4.3.1 顶部状态栏

显示：

- `18 岁`
- `2013 年`
- `学生阶段`
- 当前地区

交互：

- seed 旁有复制图标。
- 状态栏 sticky。

### 4.3.2 状态摘要

核心数值展示：

- 健康
- 财富
- 幸福
- 成就
- 名声
- 自由

使用紧凑进度条，不要大面积仪表盘。

基础属性可以折叠：

- 体质 / 智力 / 魅力 / 家境 / 运气 / 心态

### 4.3.3 事件卡片

人生推进页使用连续时间轴。每年可能出现 1-3 条事件，每条事件一张轻量卡片，并追加在同一个可滚动列表中。点击“下一年”不会切换页面，只是在当前时间轴底部追加新年份内容。

年份之间使用轻量分隔：

- 年龄 + 公历年份作为小标题。
- 不使用大块章节封面。
- 不用满屏动画打断阅读。
- 新事件出现后页面可自动轻微滚动到最新年份。

卡片内容：

- 事件标题
- 事件正文
- 状态变化摘要
- 标签变化摘要

示例：

```text
18 岁 · 高考超常发挥
你在高考中超常发挥，考上了一所不错的大学。

成就 +10   幸福 +5   获得标签：大学
```

事件类型视觉：

- 普通：默认卡片
- 关键：左侧 accent 色条
- 危机：danger 色条
- 死亡：整卡低饱和红色提示
- 结局：特殊报告样式

### 4.3.4 下一年按钮

按钮文案根据状态变化：

- 普通：下一年
- 死亡：查看人生报告
- 已结局：再来一局

交互：

- 点击后轻微过渡，事件卡逐条出现。
- 新年份事件追加到同一条时间轴底部。
- 如果玩家不在底部，不强制抢滚动；只给“有新年份”提示。
- 如果玩家已经在底部，自动跟随到最新事件。
- 不做太慢动画，点点点要顺。

## 4.4 自动结果展示

人生推进过程中不允许出现选择弹窗、确认弹窗或分叉按钮。关键事件如有多个可能结果，由引擎根据人物状态、权重和 seed 自动决定，并在同一张事件卡内展示最终结果。

结构：

- 事件标题
- 事件描述
- 自动发生的结果文案
- 属性、资源、特质和标签变化

示例：

```text
第一份工作
毕业后，你走到了第一份工作的门口。
你进入大公司，先换来了一份稳定。

财富 +8   获得标签：大公司职员
```

结果产生后：

- 当前年度事件卡直接追加结果文本。
- 同一次年度推进内应用效果并写入快照。
- 玩家继续点击固定位置的“下一年”。

## 4.5 人生报告页

目标：让玩家觉得这一局值得回味，并想分享 / 重开。

结构：

1. 结局标题
2. 最终年龄和死亡 / 结局原因
3. 一段人生总结
4. 最终评分
5. 关键时间线
6. 称号和解锁内容
7. seed 分享
8. 再来一局

### 4.5.1 结局头部

示例：

```text
你的一生结束了
享年 47 岁
结局：县城做题家的中年转弯
```

死亡可以很早发生。0 岁死亡也直接进入报告页，不额外遮挡。

### 4.5.2 时间线

只展示关键事件，不展示所有普通事件。

筛选规则：

- priority 事件
- 自动结果事件
- 获得重要标签
- 大幅数值变化
- 死亡 / 结局事件

### 4.5.3 分享区域

展示：

- seed
- 出生参数摘要
- 结局标题
- 最终评分

操作：

- 复制 seed
- 复制人生摘要
- 再来一局

## 4.6 图鉴页

首版可以简单做。

Tabs：

- 天赋
- 结局
- 称号
- 稀有事件

每个 item：

- 未解锁：显示轮廓和问号
- 已解锁：显示名称、描述、首次解锁时间

## 6. 关键状态

### 6.1 加载状态

- 初次进入：短 loading，不要复杂动画。
- 年度推进：按钮进入 loading，事件卡逐条出现。

### 6.2 空状态

- 图鉴为空：展示“还没有解锁任何结局”。
- seed 无效：展示“这个 seed 看起来不太对”。

### 6.3 死亡状态

- 人生页按钮变成“查看人生报告”。
- 不弹二次确认。
- 不隐藏极端事件。

### 6.4 错误状态

- 数据包加载失败：提示刷新。
- 事件配置错误：开发环境显示详细错误，生产环境跳过该事件并记录日志。

## 7. 响应式规则

桌面：

- 最大内容宽度：1200px。
- 人生页使用主内容 + 右侧栏。
- 设置页使用左侧步骤 + 右侧内容。

移动：

- 单列布局。
- 下一年按钮吸底。
- 状态摘要可折叠。
- 事件卡占满宽度。

### 7.1 移动端设计规格

手机端需要单独设计，不能只是桌面布局压缩。

视口基准：

- 主要设计基准：`390 x 844`
- 兼容小屏：`360 x 740`
- 平板断点：`768px`
- 桌面断点：`1024px`

布局：

- 页面左右边距：`16px`，小屏可降到 `12px`。
- 卡片圆角保持 `8px`。
- 卡片之间垂直间距 `12-16px`。
- 不使用左右双栏。
- 不出现横向滚动。
- 底部主按钮使用 sticky / fixed，距离底部安全区至少 `12px`。
- 吸底按钮上方需要留出内容 padding，避免遮住最后一张事件卡。

文字：

- 正文最小 `15px`。
- 事件标题 `20-22px`。
- 页面标题 `28-36px`。
- 报告页结局标题可以到 `40-44px`，但必须换行自然。
- 不使用随视口宽度无限缩放的字体。

顶部状态栏：

- 手机端 sticky。
- 第一行：年龄 + 当前年份 + 阶段。
- 第二行可选：当前地区 + seed 复制。
- 不展示完整属性条。
- 高度尽量控制在 `72-96px`。

人生推进页：

- 事件卡是主内容。
- 年与年在同一个可滚动时间轴内连续排列。
- 每年 1-3 条事件纵向追加。
- 年份分隔符要轻，不做整屏切换。
- 事件变化摘要用小 pill，不要占满多行。
- 状态摘要默认折叠，入口放在顶部状态栏或事件列表之后。
- 点击“下一年”后，事件卡从上到下出现。
- 玩家可以向上滑动回看过去年份，底部按钮仍可继续推进。

开局设置页：

- 使用顶部进度条替代左侧 stepper。
- 每一步单列。
- 出生设置字段使用原生 select / segmented control。
- 属性分配每行高度至少 `44px`。
- 天赋卡单列，每张卡高度固定或设置最小高度，避免跳动。
- 底部操作区：上一步、下一步、随机一下。

自动结果事件：

- 和普通事件一样直接进入时间线，不打开弹层。
- 结果文案紧跟事件正文，文案过长时正常换行。
- 玩家看完后仍只需要点击固定位置的“下一年”。

报告页：

- 首屏必须包含结局标题、最终年龄/原因、人生摘要、再来一局、复制 seed。
- 时间线放在首屏之后。
- 最终评分使用紧凑卡片或两列网格。
- 分享区按钮不超过两个主操作：复制摘要、复制 seed。

移动端验收清单：

- `390 x 844` 无横向滚动。
- `360 x 740` 不出现按钮文字挤压。
- 首页首屏能看到“开始新人生”。
- 设置页任一步都能在两次以内滚动完成。
- 人生页单手点击“下一年”顺手。
- 死亡报告页不被底部按钮遮挡。
- 事件正文和按钮文字不重叠、不溢出。

## 8. 首版页面优先级

## 10. 待视觉确认

- 最终品牌名。
- 是否使用更强游戏感的视觉，还是保持“人生档案”风格。
- 是否要在首页使用插画 / 生成图。
- 天赋稀有度是否需要更强烈的视觉反馈。
- 报告页是否需要一键生成分享长图。
