# 虚构未来史写作框架

`src/data/events/history/speculative-history.js` 写的是文学性的历史假设，不是对现实的事实断言。它的目标是让 2036—2132 年也拥有时代压力、公共冲突、制度变化与集体记忆，不让未来只剩新设备和更方便的日常生活。

## 年代骨架

| 年代 | 时代进程 | 人生中的落点 |
| --- | --- | --- |
| 2036—2044 | AI 劳动审计、时间分红、人工责任签署 | 工时归谁、错误由谁负责 |
| 2036—2052 | 高龄热浪名单、热历、夜城、公共降温权 | 工作时钟、照护和住房差异 |
| 2050—2060 | 照护缺口、照护时账户、旧学校改造 | 未计价劳动与代际关系 |
| 2058—2068 | 水配额、街区微电网、流域协议 | 资源冲突与地方互助 |
| 2064—2075 | 延缓衰老、年龄争议、公共延寿底线 | 财富能否购买更多健康时间 |
| 2070—2080 | 海岸退让、旧街告别、高地重建 | 迁移、故乡和邻里网络 |
| 2076—2085 | 数字逝者、记忆同意、终止模拟权 | 哀悼、数据与身后自主 |
| 2083—2092 | 新主食、七日断网、离线影子系统 | 脆弱的便利与笨拙的韧性 |

## 写作规则

1. 每条进程至少包含一次压力、一次社会回应和一次不完全的解决；后续事件必须依赖前因。
2. 先写制度怎样落到一顿饭、一张排班表、一次搬家或一份合同，再写技术名词。
3. 同一变化必须容纳阶层、地点、年龄和职业差异；未来不能被写成所有人同时收到的产品更新。
4. 技术不会自动带来正义。它会重新分配时间、风险、责任与被看见的资格。
5. 保留白描、克制和一点人的幽默，避免预言口吻、口号式赞美与灾难奇观。
6. 不增加任何中途选择。玩家开局以后仍只有“下一年”，时代分叉由状态和既往经历自动决定。

## 批测门禁

分层测试单列 2036—2132 年：统计虚构未来史占未来个人年的比例、被实际触发的事件定义数，并按七个时间窗检查空白。后续扩写若只增加文件数量、却无法进入实际人生或造成某个年代断层，严格审查会直接失败或进入人工审查队列。
