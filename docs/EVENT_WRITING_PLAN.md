## 主要问题

1. 事件总量仍然很少。200 条只能证明结构跑通，远远不够“人生质感”。
2. 日常事件密度不足。虽然已补一批低门槛事件，但不同年龄、城乡、性别、阶层、时代的普通年份仍会重复。
3. 历史事件覆盖太粗。很多大时代目前只有一两条代表事件，缺少地区、阶层、年龄、性别差异。
4. 职业系统仍偏薄。职业路径只有几个大入口，缺少职业内成长、挫败、转行、行业周期。
5. 家庭和关系还不够真实。缺少兄弟姐妹、赡养、婆媳/姻亲、离婚、再婚、代际冲突、亲友互助和拖累。
6. 事件链条不足。很多事件只是一次性效果，缺少后续 3-10 年的连锁。
7. 结局/死亡事件少。死亡、特殊结局、时代结局、职业结局都还不够。

## 扩写原则

1. 同一个人生语义事件保留一个事件；只是表达不同，使用 `text` variants。
2. 只有触发逻辑、后果、优先级、依赖关系或历史意义真的不同，才拆成不同事件。
3. 性别是条件之一，不是所有事件都必须体现性别。
4. 年代语境必须优先于现代直觉。1936 年不能出现“大公司 offer”，1968 年不能出现“投简历”。
5. 每批事件必须能跑模拟，不允许只写数据不验证。
6. 每个文件增长到 100-150 条后继续拆分。
7. 写事件时优先补“候选池空洞”，再补传奇大事件。

## 推荐补写顺序

1. 日常事件补到 300 条，优先解决空白和重复。
2. 第一份工作/职业路径补到 200 条，避免时代穿帮。
3. 1900-1948 与 1949-1977 各补 150 条，因为这两个时代最容易被现代语感污染。
4. 婚育/家庭/关系补 200 条，提升人生叙事连续性。
5. 改革开放、打工潮、下岗、WTO、互联网各补 80-120 条。
6. 死亡与结局补到 80 条，形成差异化终局。
